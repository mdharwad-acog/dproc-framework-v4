Contributing to DProc

Developer Guide

How to extend and improve the framework

Ways to Contribute

  • Write a new pipeline for a use case
  • Add features to the core framework
  • Fix bugs and improve errors
  • Improve documentation and tests

Creating a New Pipeline

  • Use dproc init my-pipeline
  • Define inputs in spec.yml
  • Implement processor.ts
  • Write prompts in prompts/
  • Create templates in templates/

Processor Best Practices

  • Use context.logger for debugging
  • Handle errors gracefully with try-catch
  • Return structured ProcessorResult
  • Use caching for expensive operations

Framework Development

  • Edit packages/dproc-core for core logic
  • Update types in packages/dproc-types
  • Test with pnpm test
  • Follow TypeScript strict mode

Error System

  • Use DProcError for structured errors
  • Provide user-friendly messages
  • Include error codes and context
  • Suggest fixes when possible

Input Validation

  • Validator auto-converts string inputs
  • Supports types: text, number, boolean, select, file
  • Validates file size and MIME type
  • Provides detailed error messages

Testing Your Pipeline

  • Validate structure: dproc validate
  • Test execution: dproc run my-pipeline
  • Check outputs in output/reports/
  • View bundles in output/bundles/

Development Workflow

  • Clone the repository
  • Install: pnpm install
  • Run dev: pnpm dev
  • Make changes and commit

Useful Commands

  • pnpm install - Install dependencies
  • pnpm build - Build all packages
  • pnpm test - Run test suite
  • pnpm dev - Start development server

Code Quality Standards

  • TypeScript strict mode enabled
  • ESM modules only
  • Proper error handling required
  • Logging for debugging
